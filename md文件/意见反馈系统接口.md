# 意见反馈系统接口文档

## 接口说明

本文档用于意见反馈系统第一版前后端联调。第一版只覆盖当前前端实际需要的核心链路：用户提交反馈、查看反馈、继续追问，管理员查看反馈、回复反馈、修改状态。

第一版完整流程：

```plain
用户提交反馈 -> 用户查看反馈记录 -> 用户继续追问 -> 管理员后台查看反馈 -> 管理员回复/改状态 -> 用户前台看到处理结果
```

统一响应格式参考现有后端文档：

```plain
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

错误响应示例：

```plain
{
  "code": 400,
  "message": "参数错误",
  "data": {}
}
```

# 用户反馈模块

## 查询我的反馈列表

### 基本信息

+ 接口说明：用户进入意见反馈页面时，查询当前登录用户自己的反馈记录和消息列表。
+ 请求路径：`feedback/user/tickets`
+ 请求方式：GET

### 请求参数

参数格式：query string

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20 |

请求示例：

```plain
feedback/user/tickets?page=1&pageSize=20
```

### 响应数据

参数格式：application/json

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| code | number | 是 | 状态码 |
| message | string | 否 | 提示信息 |
| data | object | 是 | 返回数据 |
| |-tickets | array | 是 | 反馈列表 |
| |-total | number | 否 | 总数量 |
| |-page | number | 否 | 当前页码 |
| |-pageSize | number | 否 | 每页数量 |

ticket 字段说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 反馈 ID |
| type | string | 是 | 反馈类型 |
| status | string | 是 | 反馈状态 |
| subject | string | 是 | 反馈标题 |
| content | string | 是 | 初始反馈内容 |
| createdAt | string | 是 | 创建时间 |
| updatedAt | string | 是 | 更新时间 |
| messages | array | 是 | 消息列表 |

message 字段说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 消息 ID |
| body | string | 是 | 消息内容 |
| isAdmin | boolean | 是 | 是否管理员消息 |
| createdAt | string | 是 | 创建时间 |
| authorName | string | 是 | 发送人名称 |

响应数据示例：

```plain
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "tickets": [
      {
        "id": "ticket-1",
        "type": "generation",
        "status": "open",
        "subject": "生成结果亮度不稳定",
        "content": "同样提示词第二次生成偏暗",
        "createdAt": "2026-07-09T10:24:00.000Z",
        "updatedAt": "2026-07-09T10:46:00.000Z",
        "messages": [
          {
            "id": "message-1",
            "body": "同样提示词第二次生成偏暗",
            "isAdmin": false,
            "createdAt": "2026-07-09T10:24:00.000Z",
            "authorName": "Mina"
          }
        ]
      }
    ],
    "total": 1,
    "page": 1,
    "pageSize": 20
  }
}
```

### 备注说明

+ 该接口必须登录。
+ 只能返回当前登录用户自己的反馈，不能返回其他用户的数据。

## 提交反馈

### 基本信息

+ 接口说明：用户创建一条新的意见反馈。
+ 请求路径：`feedback/user/tickets`
+ 请求方式：POST

### 请求参数

参数格式：application/json

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| type | string | 是 | 反馈类型 |
| subject | string | 是 | 反馈标题 |
| content | string | 是 | 反馈详细内容 |

请求数据示例：

```plain
{
  "type": "generation",
  "subject": "生成结果亮度不稳定",
  "content": "同样提示词第二次生成偏暗，希望确认是否是默认曝光策略导致。"
}
```

### 响应数据

参数格式：application/json

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| code | number | 是 | 状态码 |
| message | string | 否 | 提示信息 |
| data | object | 是 | 返回数据 |
| |-ticket | object | 是 | 新创建的反馈 |

响应数据示例：

```plain
{
  "code": 200,
  "message": "反馈提交成功",
  "data": {
    "ticket": {
      "id": "ticket-1",
      "type": "generation",
      "status": "open",
      "subject": "生成结果亮度不稳定",
      "content": "同样提示词第二次生成偏暗，希望确认是否是默认曝光策略导致。",
      "createdAt": "2026-07-09T10:24:00.000Z",
      "updatedAt": "2026-07-09T10:24:00.000Z",
      "messages": [
        {
          "id": "message-1",
          "body": "同样提示词第二次生成偏暗，希望确认是否是默认曝光策略导致。",
          "isAdmin": false,
          "createdAt": "2026-07-09T10:24:00.000Z",
          "authorName": "Mina"
        }
      ]
    }
  }
}
```

### 备注说明

+ 该接口必须登录。
+ `type` 必须是后端允许的枚举值。
+ `subject` 和 `content` 不能为空。
+ 建议 `subject` 最大长度为 120 字符。
+ 建议 `content` 最大长度为 5000 字符。
+ 用户提交反馈时，建议同时在消息表中写入一条用户消息，方便后续形成对话记录。

## 用户继续追问

### 基本信息

+ 接口说明：用户对已有反馈继续补充信息。
+ 请求路径：`feedback/user/tickets/{ticketId}/messages`
+ 请求方式：POST

### 请求参数

参数格式：application/json

路径参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| ticketId | string | 是 | 反馈 ID |

Body 参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| body | string | 是 | 补充内容 |

请求数据示例：

```plain
{
  "body": "我补充一张截图，问题仍然存在。"
}
```

### 响应数据

参数格式：application/json

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| code | number | 是 | 状态码 |
| message | string | 否 | 提示信息 |
| data | object | 是 | 返回数据 |
| |-ok | boolean | 是 | 是否成功 |

响应数据示例：

```plain
{
  "code": 200,
  "message": "发送成功",
  "data": {
    "ok": true
  }
}
```

错误响应示例：

```plain
{
  "code": 403,
  "message": "无权操作该反馈",
  "data": {}
}
```

### 备注说明

+ 该接口必须登录。
+ 必须校验该反馈属于当前登录用户。
+ `closed` 状态不能继续追问。
+ `body` 不能为空。

## 管理员回复反馈

### 基本信息

+ 接口说明：管理员对用户反馈进行回复，回复内容会回显到用户端反馈详情中。
+ 请求路径：`feedback/admin/tickets/{ticketId}/reply`
+ 请求方式：POST

### 请求参数

参数格式：application/json

路径参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| body | string | 是 | 管理员回复内容 |

请求数据示例：

```plain
{
  "body": "我们已经收到，会排查这次生成任务。"
}
```

### 响应数据

参数格式：application/json

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| code | number | 是 | 状态码 |
| message | string | 否 | 提示信息 |
| data | object | 是 | 返回数据 |
| |-ok | boolean | 是 | 是否成功 |

响应数据示例：

```plain
{
  "code": 200,
  "message": "回复成功",
  "data": {
    "ok": true
  }
}
```

### 备注说明

+ 该接口必须登录。

+ `body` 不能为空。
+ 如果当前反馈状态是 `open`，管理员回复后建议自动变成 `in_progress`。

# 枚举说明

## 反馈类型 type

| 值 | 含义 |
| --- | --- |
| bug | 问题反馈 |
| billing | 订单与积分 |
| generation | 图片生成 |
| account | 账户问题 |
| suggestion | 功能建议 |
| other | 其他 |

## 反馈状态 status

| 值 | 含义 | 说明 |
| --- | --- | --- |
| open | 待处理 | 用户新提交，管理员尚未处理 |
| in_progress | 处理中 | 管理员已回复或开始处理 |
| resolved | 已解决 | 问题已处理完成 |
| closed | 已关闭 | 关闭后用户不可继续追问 |

# 联调验收流程

1. 普通用户提交一条反馈。
2. 普通用户能在自己的反馈记录中看到这条反馈。
3. 管理员后台能看到这条反馈。
4. 管理员回复用户。
5. 普通用户前台能看到管理员回复。
6. 普通用户继续追问。
7. 管理员后台能看到用户追问。
8. 管理员修改状态为 `resolved` 或 `closed`。
9. 如果状态是 `closed`，普通用户不能继续追问。
